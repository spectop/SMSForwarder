# 项目目标

## 1. 愿景

构建一个稳定、可配置、可扩展的 Android 短信监控与转发程序，重点服务验证码和业务通知类短信处理场景。

## 2. 阶段目标

### P0（可用版本）

1. 前台服务持续监听短信。
2. 支持至少一种匹配规则（关键词/正则）。
3. 支持至少一种提取规则（正则提取验证码）。
4. 支持 `sms_agent` 推送。
5. YAML 配置可加载并生效。

### P1（增强版本）

1. 支持 `curl` 推送。
2. 支持多工作流并行执行。
3. 支持失败重试与基本统计。
4. 提供配置校验提示。

### P2（稳定版本）

1. 开机自启与异常恢复完善。
2. 增加回放测试能力（历史样例驱动验证规则）。
3. 增加可观测性（执行日志、错误分类、耗时统计）。

## 3. 验收标准

1. 连续运行 72 小时无崩溃。
2. 规则命中准确率达到预期（由测试样例集验证）。
3. 推送成功率在网络正常情况下 > 99%。
4. 配置错误能够被检测并给出明确提示。

## 4. 非目标（当前阶段）

1. 不做运行时动态插件市场。
2. 不做复杂云端配置中心。
3. 不做跨设备配置自动同步。
