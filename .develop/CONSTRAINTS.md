# 约束清单

## 1. 技术约束

1. 开发语言为 Kotlin。
2. 运行环境为 Android，需遵循后台执行限制与权限模型。
3. 配置主存储使用 YAML 文件。
4. 推送适配器通过代码静态注册，不支持动态插件加载。

## 2. 安全与隐私约束

1. 必须显式提示并申请短信读取相关权限。
2. 默认不长期保存完整短信正文。
3. 日志中涉及敏感字段必须脱敏。
4. 网络推送建议使用 HTTPS。
5. Token/密钥等凭据不得硬编码在仓库中。

## 3. 可靠性约束

1. 每个工作流执行应具备超时与异常隔离。
2. 推送失败应可重试且不阻断其他工作流。
3. 应避免重复推送（支持幂等键或短时去重策略）。

## 4. 可维护性约束

1. 规则与业务逻辑分离，规则由配置驱动。
2. 新增推送方式时必须包含：
   - 适配器实现
   - 配置模型定义
   - 参数校验
   - 单元测试
3. 文档变更应与功能变更同步更新。
